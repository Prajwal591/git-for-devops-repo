git config: Configures Git settings for the user or repository.

git init: Initializes a new Git repository in the current direc
tory.

git clone: Creates a copy of an existing Git repository.

git add: Adds changes in the working directory to the staging area. 

git commit: Records staged changes permanently in the repository's history.

git status: Shows the status of the working directory and staging area.

git diff: Shows changes between commits, or between the working directory and the staging area.

git branch: Lists, creates, or deletes branches.

git checkout: Switches branches or restores working tree files.

git merge: Combines changes from one branch into another.

git reset: Undoes changes by moving the HEAD pointer to a different commit.

git rm: Removes files from the working directory and stages the removal.

git log: Displays the commit history.

git show: Shows information about a Git object (commit, tag, tree, or blob).

git tag: Creates, lists, deletes, or verifies a tag object signed with GPG.

git remote: Manages remote repositories.

git push: Uploads local branch commits to a remote repository.

git pull: Fetches changes from a remote repository and merges them into the current branch.

git stash: Temporarily saves changes that are not ready to be committed.

git rebase: Reapplies commits on top of another base tip.
